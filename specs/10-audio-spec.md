# 10 — Audio Spec

## Goals

- immediate feedback
- synthetic retro style
- short punchy sounds
- audio failure should not break the game

## Categories

- player fire
- hit confirm
- enemy destroyed
- player damage
- game over
- restart or start cue
- optional ambient hum later

## Browser requirements

- audio initialization must respect browser user-interaction requirements
- pointer-lock/start interaction can also enable audio
- if loading fails, gameplay still proceeds

## Style

Think electronic, arcade, concise, and readable rather than realistic.
